# 交接紀錄 HANDOVER

給下一位接手的人（不論是人類還是 AI 模型）。
技術規格請看 [`../AGENTS.md`](../AGENTS.md)，這份只講**背景、決策理由與現況**。

## 一、專案由來

專案擁有者是一位台灣的老師，需求原話：

> 「我想做一個從台灣的國小、國中、高中、大學，所有在台灣學校的所有考試題目以及解答，
> 會讓我選擇程度、學校、所有科目，然後再去出題，最後才有解答部分……
> 我想要變成能夠與人分享（手機和電腦都能用）」

後續追加需求：

> 「大學科目太少，我需要的是所有系別相關的科目都要出現。考題新增更新就會一直換題目。
> 要出現隨機搶答等關卡增加寫作難度。」

## 二、關鍵決策與理由（不要輕易推翻）

| 決策 | 理由 |
|------|------|
| **題目全部原創，不是各校真題** | 全台各校段考題沒有公開資料庫，且有著作權。原創題依 108 課綱編寫，法律安全且品質可控。網站頁尾與 README 都有明確聲明。 |
| **不做「選學校」功能** | 使用者原本要求「選學校」，但既然題目非各校真題，用學校當篩選條件會誤導使用者。改以「學段 → 科目 → 年級 → 難度」分類。**如果之後要加，請先想清楚題目來源怎麼對應到學校。** |
| **不接 AI 即時出題** | 需要 API 金鑰、每次出題都要付費，分享給別人時對方也得自備金鑰，不符合「隨手分享」的核心需求。 |
| **純靜態、無後端** | GitHub Pages 免費、免維運、免登入，網址一貼就能用。使用者資料存 localStorage。 |
| **每檔 20 題、難度 7/7/6** | 讓「闖關模式」每關（各難度）都抽得到 5 題，且「混合」模式難度分布均衡。 |
| **答錯的題目自動進錯題本** | 這是教學上最有效的複習機制，也是使用者最該推薦給學生用的功能。 |

### 題庫怎麼產出來的
用平行 subagent 扮演各科教師分頭出題（每位負責 6～9 個科目檔），
每位出題者被要求**逐題重算驗證答案**，並在寫完後自檢 JSON。
最後由主流程跑 `node validate.js` 統一把關，並人工抽查數學/物理/統計等計算題。

**這個做法的已知副作用**：平行寫入同一目錄會產生檔名不符規格的殘留檔。
第二批出題時就發生過 6 個（重複的線代/熱力學檔、拼字不同的民法檔、
以及三個規格外但有價值的新科目：演算法、刑法概要、公共行政）。
處理方式與防範機制已寫進 `AGENTS.md` 第七節。
`validate.js` 現在會主動警告「多餘檔案」。
